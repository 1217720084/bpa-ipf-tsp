With some minor revisions, the complete Linux update installation process 
follows, based on the 26-Mar-2001 update.  Future updates would follow the
same process.

1)-- IPF runs on many platforms, including Linux.  Updates include the
name of the platform at the end of the update filename.

2)-- Copy the downloaded file 26-Mar-2001.tar.gz.linux into the
"/ipfupdates" directory, e.g., /ipfupdates but not to /shr.

3)-- Rename this file from 26-Mar-2001.tar.gz.linux to 26-Mar-2001.tar.gz.

Unzip it and untar it.
 
gunzip 26-Mar-2001.tar.gz
tar -xvf 26-Mar-2001.tar

4)-- Here is the important new step, which you need to do only once,
for the 26-March-2001 update, unless future updates require modification 
of the ".bash_profile" of user login directories.  In your .bash_profile
file in your default user login directory, define the new environment 
variables before the next step.  

A)FIRST ALTERNATIVE METHOD -- EDIT EXISTING .BASH_PROFILE -------------------
Add the following line directly following the IPFDIRS="/shr/ipf-3.27/dat" line:

IPFROOTDIR="/shr/ipf-3.27"

So it looks like this:
 
IPFDIRS="/shr/ipf-3.27/dat"
IPFROOTDIR="/shr/ipf-3.27"
...
Also, append IPFROOTDIR to the “EXPORT” statement:

So it looks like this:
EXPORT IPFDIRS IPFROOTDIR 
or like this:
export UIDPATH IPFDIRS RUN_IPFSRV IPFSRV_CF IPF_SOCK_PATH IPFROOTDIR

B)SECOND ALTERNATIVE METHOD -- REPLACE EXISITNG .BASH_PROFILE -----------

Download a copy of the file: "sample.bash_profile" from the BPA IPF FTP Site 
or copy the text of "sample.bash_profile" from the bottom of this file.

Replace the ".bash_profile" in the /root directory, in the /home/ipf327user 
directory, and in the login directory of any user who will be running the
IPF Programs, with the contents of "sample.bash_profile".

5)-- Build the new IPF programs with the command
 
      cd /ipfupdates/26-Mar-2001
      make

This should run to completion.  

------------------------------------------------------------------------------

TEXT OF THE FILE "SAMPLE.BASH_PROFILE"

# .bash_profile modified for use with IPF-3.27.17 [26-March-2001 update]
# "root" user MUST also replace the default .bash_profile initialization file 
#  with this new version.
# File:  /root/.bash_profile    
# Syntax:  "cp ./sample.bash_profile ./bash_profile"

# Get the aliases and functions
if [ -f ~/.bashrc ]; then
	. ~/.bashrc
fi

# User specific environment and startup programs

PATH=$PATH:$HOME/bin
ENV=$HOME/.bashrc
USERNAME=""

export USERNAME ENV PATH

mesg n

UIDPATH="/shr/ipf-3.27/exe/%U:/usr/lib/X11/uid/%U"
# path to locate help and sample data files
IPFDIRS="/shr/ipf-3.27/dat/"  
IPFROOTDIR="/shr/ipf-3.27"       
# optional alternative IPFDIRS=<IPF base case directory>
IPFSRV="/shr/ipf-3.27/exe/ipfsrv"
IPFSRV_CF="/shr/ipf-3.27/exe/ipfsrv" 
# IPF_SOCK_PATH=<IPF temporary directory to hold socket locking files>
IPF_SOCK_PATH="/shr/ipf-3.27/temp"
RUN_IPFSRV="/shr/ipf-3.27/exe/ipfsrv"

export UIDPATH IPFDIRS RUN_IPFSRV IPFSRV_CF IPF_SOCK_PATH IPFROOTDIR

# /usr/games/fortune

# startx
